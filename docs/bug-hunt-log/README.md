# Bug hunt log

Daily, automated audit of the repo. Generated by
`scripts/bug-hunt/run.mjs` and scheduled in
`.github/workflows/daily-bug-hunt.yml` at **00:00 UTC** every day.

Each run produces a markdown file `YYYY-MM-DD.md` with three sections:

1. **Checks** — pass/fail/skipped status for `lint`, `typecheck`, and
   `tests` (vitest run). Failed checks include the last 12 lines of
   their output.
2. **Pattern findings** — counts and details for each static pattern
   defined in `scripts/bug-hunt/patterns.mjs`. Every pattern is derived
   from a real defect that was previously found and fixed; the daily
   scan keeps the same shape from coming back.
3. **Fixed since `<prior date>`** — patterns whose hit count dropped
   versus the previous log. This is the running "bugs fixed" record.

## Running locally

```bash
node scripts/bug-hunt/run.mjs            # write today's log
node scripts/bug-hunt/run.mjs --dry-run  # print to stdout
BUG_HUNT_SKIP_NPM=1 node scripts/bug-hunt/run.mjs  # patterns only
BUG_HUNT_DATE=2026-05-12 node scripts/bug-hunt/run.mjs  # backfill
```

## Adding a pattern

1. After fixing a bug, write a scanner in
   `scripts/bug-hunt/patterns.mjs` that flags the *shape* of the
   defect — not the specific file or line, so the check catches the
   next occurrence wherever it lands.
2. Add a one-paragraph block comment to the scanner that links to the
   log entry that motivated it (date + issue, e.g. `2026-05-12 H1`).
3. Append a row to the `PATTERNS` export with a stable `id`,
   `severity` (`high|medium|low`), `title`, and the `scan` function.

The `id` is durable — the daily diff uses it to compute "fixed since
<date>", so renaming an id wipes the historical count for that pattern.

## Where the logs live

- **Per run** — uploaded as an `actions/upload-artifact@v4` artifact
  named `bug-hunt-YYYY-MM-DD` (90-day retention by default).
- **Long-term** — committed to the `bug-hunt-logs` branch under
  `docs/bug-hunt-log/`. `main` does not accumulate logs.
- **High-priority surface** — a GitHub issue titled
  `Daily bug hunt — YYYY-MM-DD` is opened or updated when the run
  finds any HIGH-severity hit or any failing check. The issue is
  upserted by title so daily runs don't spam.
